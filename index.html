Input Signals

The ALU is a ripple carry adder with a few extra options.
A/~A B/~B Carry-In Flood-Carry OR instead
of XOR		 Function
0 0 0 0 0 ADD
(A+B)
0 1 1 0 0 SUB
(A-B)
1 0 1 0 0 SUB
(-A+B)
0 0 0 1 1 NOR
(~[A|B])
1 1 0 1 1 AND
(A&B)
0 0 0 0 0 XOR
(A^B)
1 0 0 1 0 XOR
(A^B)
Other combinations are rarely used
  • A/~A / ~A: If A (Input) should be inverted bitwise
  • B/~B / ~B: If B (Akku) should be inverted bitwise
  • Carry-In / c-in: If there should be a Carry-In
  • Flood-Carry / flca: Forces all carrys to be 1
  • OR/XOR / usOR: Use OR instead of XOR with inputs Ai and Bi
    Does not affect the combination with the Carry-In-Bit of the Full-Adder.
  • writeAk / wrAk: If the ALU result should be saved to the Akkumulator and if the Carry-Out should be saved
  • shAk: If the Akkumulator should be shifted, the Shift-Counter will be decremented if this is true
  • shMQ: If the MQ-Register should be shifted, the Shift-Counter will be decremented if this is true
  • ResetMQ / rsMQ: If the MQ-Register should be set to 0
  • shl/shr / sh-L: The direction of the Shift of the Akkumulator or MQ-Register
  • ResetAk / rsAk: If the Akkumulator should be set to 0
  • 1->MQ0 / sMQ0: If the last bit of the MQ-Register should be set to 1
  • ResetSC / rsSC: If the Shift-Counter should be set to 0

Output Signals

  • Carry-Out / c-o: The Carry-Out signal of the last saved Addition. Indicates over- and underflow
  • Sign / sgn: The most significant bit of the Akkumulator
  • Ak=0 / ak0: If the Akkumulator holds the value 0
  • SC=0 / sc0: If the Shift-Counter holds the value 0
  • MQ0 / mq0: The least significant bit of the MQ-Register

Instructions

A Program is json-file with this schema.

All instructions have a type: mem, jmp or alu.

All instructions can have a comment field.

Memory Instructions (mem)

Write to the Input-Register and/or the Shift-Counter. Values are either integers or for the Input-Register only: "in1" or "in2" to use the value form the User-Inputs.

Jump Instructions (jmp)

Used to implement if- and while-loops. Change the Program-Counter based on a Signal.

If a jump is not taken, the Program-Counter increments by 1 like normal.

  • signal: The Output Signal that gives the decision. jmp is a special signal for unconditional jumps.
  • if: If the jump should be taken if the signal is on or off. Is required for all signals but jmp where it does nothing.
  • skip: Jump relatively to the current line. If positive, skip skip lines. If negative, go skip lines back.
  • line: Go to this line. Line numbers are absolute.

You need to provide either skip or line. Providing both will first jump to the absolute line and then jump relative to that.

ALU Instructions

ALU Instructions set the Input Signals and perform shifts and saving of the ALU result.

All signals are optional and default to false / off.

Custom Programs

To create a custom Program, download the schema and create a new .json-file.
Include "$schema": "<path/to>/program.schema.json" to make your editor guide you and validate the schema.
Give you program a name using the name property.
Specify what circuit you program is for using the unit property.
Put your instructions into the instructions array. Each instruction is an object that specifies the the instruction-type in the type-property and it's options.

The state is randomized on site load. It is recommended to include this as your first instruction:
{ "type": "alu", "rsAk": true, "rsMQ": true, "rsSC": true}
This resets all registers. The input signals do not need to be reset. Only the Carry-Out is not reset.

UI

On the left you have circuit with its current state. And on the right you have the program being executed. Both come with their own toolbar.

Running Programs

To run a program you can step through it using the Step-button. Alternatively you can hit the Play-button to let the program run at the speed specified by the slider. To stop it, hit the Play-button again. To start at the start again, hit the Reset-button, which sets resets the program counter.

You can also set breakpoints, where, once you hit play, it stops. To set a breakpoint, click the line-number of the line you want to break before it executes. To remove a breakpoint, click again.

Changing the program also resets the Program-Counter. If the program executes HALT, it stops.

User Input

Apart from choosing the executing circuit and program you can also provide user input to the program by using the two input fields. Numbers are entered in decimal and the program needs to be instructed to read it. The read-only Output-Field shows the value of the Akkumulator once the program halts in decimal.
